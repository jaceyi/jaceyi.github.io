前言

本篇讲述 React Class 组件中 setState 方法在不同情况下“异步”、同步执行，以及其带来的影响。

“异步”执行

初学 React 的同学或许会碰到下面的问题。在 setState 后数据并没有实时更新，取到的数据还是老数据。

  
state = {
  num: 0
}

handleClickAdd() {
  this.setState({
    num: this.state.num + 1
  });
  console.log(this.state.num) // 0
}

在上面输出的 num 还是为 0，这是否代表 setState 是异步操作呢
